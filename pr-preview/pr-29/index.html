clawdputer / web demo

Browser simulation of the firmware in CHaerem/clawdputer. The 240×135 display is a real pixel buffer; the keyboard is wired to your physical keys and to clickable on-screen keys.

CARDPUTER

How to drive it

  • , / / — rotate cards in the launcher
  • ; . , / — arrow keys in menus (↑ ↓ ← →)
  • Enter — launch / confirm / send
  • Tab or G0 — back to home from any app
  • Backspace — delete / cancel
  • Type to enter text in Chat & SSH inputs

What you're looking at

  • Buddy — the Claude Desktop companion screen. After a moment it goes "connected" and a permission prompt arrives so you can press Y / N.
  • Chat — the device-side UI for a claude CLI session spawned by the Mac-side bridge. Streamed responses are simulated.
  • SSH — sealed host-preset picker.
  • Settings — the same sections the real device shows.

About the simulation

The display is a 240×135 canvas matching the real IPS panel one-to-one. Apps are reimplemented in JavaScript with the same layouts and key bindings as the firmware, but BLE, WiFi, the bridge daemon, and the claude CLI are stubbed with scripted activity.

Memory lives in your browser. Chat history, SSH hosts you add, Buddy approve/deny counts, and your last-selected card all persist in localStorage and survive a page reload. Wipe state any time from Settings → reset demo memory or with the button below.

Firmware & sources: CHaerem/clawdputer. Built for the M5Stack Cardputer ADV.
